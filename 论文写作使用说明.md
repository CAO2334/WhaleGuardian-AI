# 论文写作使用说明

本说明用于指导如何配合以下 3 份文档，让另一个 AI 协助完成毕业论文写作：

- `THESIS_WRITING_PACKET.md`
- `THESIS_PROMPTS.md`
- `THESIS_FIGURES_TABLES.md`

目标是让另一个 AI：

1. 不编造数据；
2. 严格依据现有实验结果写作；
3. 能按章节分别生成论文内容；
4. 能正确引用图表、CSV、JSON 和图片路径。

---

## 一、这 3 份文档分别是做什么的

### 1. `THESIS_WRITING_PACKET.md`

作用：

- 提供事实层面的全部写作材料；
- 包括项目背景、数据集统计、训练结果、消融实验、最终最佳模型、图表路径、章节框架等；
- 是整个论文写作的“事实主文档”。

可以理解为：

```text
它负责告诉另一个 AI：这个项目到底做了什么、结果是什么、哪些文件能引用。
```

### 2. `THESIS_PROMPTS.md`

作用：

- 提供“怎么让另一个 AI 写某一章”的现成提示词；
- 包括写摘要、写绪论、写第三章、写第五章、写系统实现、写整篇论文初稿等。

可以理解为：

```text
它负责告诉另一个 AI：现在你具体要写哪一部分。
```

### 3. `THESIS_FIGURES_TABLES.md`

作用：

- 提供论文中所有建议图表的编号、标题、来源路径、插入章节和建议说明；
- 便于另一个 AI 在写作时知道“这张图应该放哪、叫什么、为什么放这里”。

可以理解为：

```text
它负责告诉另一个 AI：图表该怎么编号、怎么引用、放在论文哪个位置。
```

## 二、推荐的使用顺序

最推荐顺序如下：

### 第一步：先给 `THESIS_WRITING_PACKET.md`

作用：

- 先让另一个 AI 建立项目上下文；
- 理解任务类型、最终模型、实验结论和数据来源。

建议你先发给另一个 AI：

```text
请先完整阅读 THESIS_WRITING_PACKET.md，并只基于其中提供的事实理解本项目，不要开始写作，先建立统一项目上下文。
```

### 第二步：再给 `THESIS_FIGURES_TABLES.md`

作用：

- 让另一个 AI 知道后面写章节时该插入哪些表和图；
- 便于它在正文里提前预留“表 3-1”“图 5-4”这类位置。

建议你接着发：

```text
请继续阅读 THESIS_FIGURES_TABLES.md，理解论文中建议使用的图表编号、标题、来源路径和插图位置。后续写作时必须与该文件保持一致。
```

### 第三步：最后从 `THESIS_PROMPTS.md` 里选一个提示词

作用：

- 开始真正写某一部分；
- 比如摘要、第三章、第五章、整篇初稿等。

这一步才是真正开始生成正文。

## 三、最稳的实际操作流程

如果你准备让另一个 AI 写论文，建议按下面流程来。

### 方案 A：按章节逐步写

适合你想精细控制、逐章修改。

顺序建议：

1. 给 `THESIS_WRITING_PACKET.md`
2. 给 `THESIS_FIGURES_TABLES.md`
3. 从 `THESIS_PROMPTS.md` 里选择“写中文摘要”
4. 再选择“写第一章 绪论”
5. 再选择“写第三章 数据集构建与预处理”
6. 再选择“写第四章 模型设计”
7. 再选择“写第五章 实验设计与结果分析”
8. 再选择“写第六章 系统设计与实现”
9. 再选择“写第七章 总结与展望”

优点：

- 每章都可以人工校对后再继续；
- 不容易出现整篇上下文漂移；
- 适合毕业论文这种需要反复修改的任务。

### 方案 B：先生成整篇初稿，再逐章返工

适合你想先有一版完整成文。

顺序建议：

1. 给 `THESIS_WRITING_PACKET.md`
2. 给 `THESIS_FIGURES_TABLES.md`
3. 从 `THESIS_PROMPTS.md` 里使用“生成整篇论文初稿”
4. 生成后再逐章校对、重写和润色

优点：

- 能很快得到完整论文骨架；
- 后续只需重点修正质量不高的章节。

缺点：

- 一次生成过长时，另一个 AI 容易在细节上出错；
- 图表编号一致性可能不如分章生成稳定。

对你目前这个项目，推荐：

```text
优先使用方案 A：按章节逐步写。
```

## 四、建议优先写哪些章节

如果时间有限，建议按下面顺序：

1. 中文摘要
2. 第三章 数据集构建与预处理
3. 第五章 实验设计与结果分析
4. 第六章 系统设计与实现
5. 第七章 总结与展望
6. 第一章 绪论
7. 第二章 相关理论与关键技术
8. 英文摘要

原因：

- 第三、五、六章最依赖你现在已经跑出的真实实验与系统结果；
- 这几章最容易写出“像真的完成了项目”的论文内容；
- 第二章和第一章相对更通用，可以后补。

## 五、让另一个 AI 写作时必须强调的规则

每次开始新章节前，建议都补一句约束：

```text
请严格依据 THESIS_WRITING_PACKET.md 和 THESIS_FIGURES_TABLES.md 中提供的事实、数值、路径和图表信息写作，不允许虚构实验数据、模型结果、系统功能或参考结论。
```

再补一句风格约束：

```text
正文必须直接写成论文作者口吻，采用“本文/本研究”陈述事实与结论，禁止出现“根据资料包”“根据项目资料”“文档显示”“图表目录指出”“配置文件表明”等元叙述。
```

如果写的是实验章节，再补一句：

```text
所有 Accuracy、Macro F1、类别数、训练集/验证集规模、最终模型选择理由，必须与 THESIS_WRITING_PACKET.md 一致。
```

如果写的是系统章节，再补一句：

```text
当前默认部署模型是 artifacts/final_model_04，请不要写成旧版 v1 模型。
```

## 六、写完后怎么校对

另一个 AI 生成章节后，建议按这 4 步人工校对。

### 1. 对照 `THESIS_WRITING_PACKET.md` 检查事实

重点看：

- 模型名是否正确
- 最终最佳研究模型是否写成 `04_transformer_mean_focal_mixup_cutout`
- baseline 是否写成 `01_resnet50_ce_plain`
- Accuracy / Macro F1 数值是否一致
- 数据集规模、类别数、训练/验证集样本数是否一致

### 2. 对照 `THESIS_FIGURES_TABLES.md` 检查图表编号

重点看：

- 表 3-1、表 5-1、图 5-4 这类编号是否前后一致
- 图表标题是否合理
- 图表路径和用途是否对应

### 3. 检查是否出现“申请书口吻”而不是“论文口吻”

需要删掉这类表达：

- “本项目拟……”
- “计划实现……”
- “预计达到……”
- “根据资料包……”
- “根据项目资料显示……”
- “项目资料已明确给出……”
- “根据图表目录……”
- “由 config.json 可知……”

应该改成：

- “本文完成了……”
- “实验结果表明……”
- “系统实现了……”
- “模型采用了……”
- “实验设置为……”

### 4. 检查是否出现未实现内容被写成已实现

例如：

- 当前实际 Web 后端是 Flask，模型部署使用 ONNX Runtime
- 不要写成未实际完成的其他技术栈
- 如果前端只是演示页面，也不要夸大成完整商业化系统

## 七、推荐的章节交付格式

为了方便后续粘贴到 Word 里，建议让另一个 AI 输出时遵守：

```text
1. 按论文章节编号输出
2. 使用自然段，不用项目符号堆砌
3. 图表位置用“[在此处插入图 X-X：……]”保留
4. 不输出 Markdown 代码块
5. 保持正式学术语气
```

你可以附加一句：

```text
请直接输出适合粘贴到毕业论文 Word 文档中的正文格式，不要使用 Markdown 列表或代码块。
```

## 八、建议你自己最后统一检查的 10 个点

1. 论文题目是否统一
2. 最终模型是否统一写为 `04_transformer_mean_focal_mixup_cutout`
3. baseline 是否统一写为 `01_resnet50_ce_plain`
4. 数据集规模是否统一为 51033 / 28 类
5. 训练/验证集是否统一为 40639 / 10394
6. 指标是否统一引用 Accuracy 和 Macro F1
7. 图表编号是否统一
8. Web 默认部署模型是否写为 `artifacts/final_model_04`
9. 没做的内容是否没有被夸大
10. “未来工作”是否和“已完成工作”明确区分

## 九、最推荐的实际执行方式

你可以直接按下面这套顺序操作：

### 第一步

把这两份文件发给另一个 AI：

- `THESIS_WRITING_PACKET.md`
- `THESIS_FIGURES_TABLES.md`

并说：

```text
请先阅读这两份文档，只建立项目上下文，不开始写作。
```

### 第二步

从 `THESIS_PROMPTS.md` 里复制一个提示词，例如“写第三章 数据集构建与预处理”。

### 第三步

让它输出某一章正文后，你人工校对：

- 数据对不对
- 图表编号对不对
- 语气是不是论文风格

### 第四步

逐章完成后，再让它生成：

- 中文摘要
- 英文摘要
- 总结与展望

最后统一排版。

## 十、最终一句话建议

最稳的方式不是一次让另一个 AI 写完整篇，而是：

```text
先喂 THESIS_WRITING_PACKET.md + THESIS_FIGURES_TABLES.md 建立上下文，
再用 THESIS_PROMPTS.md 逐章生成，
最后你按本说明做事实校对和图表校对。
```

这样最不容易出错，也最适合毕业论文这种需要严谨性的文稿。
